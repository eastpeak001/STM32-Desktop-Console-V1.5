# Hardware Debug Notes

## 1. 3V3 和 GND 网络短路问题

在 PCB 设计过程中曾出现 3V3 与 GND 被合并的问题。原因是误用了电源符号，将类似 GND 形状的符号改名为 3V3，导致网络混乱。

最终解决方案：

- GND 使用真正的 GND 符号。
- 3V3、5V_BUS、5V_IN、5V_PROT 全部使用 NetLabel 网络标签。
- 检查网络树，确保 3V3 与 GND 为并列网络。
- 检查 PCB 中任意 GND 焊盘，确认 Net 为 GND，而不是 3V3。

## 2. 蓝牙中文乱码问题

BT24 蓝牙串口返回中文时，在部分手机蓝牙串口助手中出现乱码。

最终解决方案：

- 蓝牙命令和回复统一使用英文 ASCII。
- 中文仅用于菜单显示或文档说明。
- 手机端发送命令时建议带换行符 \n 或 \r\n。

## 3. I2C 上拉问题

OLED、AHT20、MPU6050 模块多数自带 I2C 上拉电阻。

PCB 上预留 4.7k 上拉电阻，但默认可不焊。

如果 I2C 通信不稳定，可尝试焊接 R_SCL_PULLUP 和 R_SDA_PULLUP。

## 4. BT24 天线禁铜

BT24 蓝牙模块天线区域下方需要禁铜。否则可能影响通信距离。

设计建议：

- 天线端朝 PCB 外侧。
- 天线下方不要铺铜。
- 天线附近尽量不要放置螺丝孔、铜柱或大面积金属。
